# Easy Mode Decision Interface Spec

## Purpose

Easy Mode is a low-interaction decision interface for executive dysfunction and overwhelmed users.

It helps Cole make one decision at a time without hiding important risk.

## Core Flow

- Show one problem at a time.
- Give four choices: A/B/C/D.
- Show ATLAS recommendation.
- Explain each option in small info boxes.
- Ask simple Yes/No confirmation.
- Swipe away after each choice.
- Log the decision.
- Move to the next problem.

## Example Decision

Problem: ElevenLabs credits are running low.

A. Pause experiments.

B. Buy more credits.

C. Switch to text-only scripts.

D. Use credits only for client work.

ATLAS recommendation: D first, A second.

Why: Client and revenue work should be protected before experiments. Buying more credits is a money decision and needs approval.

## Rules

- Easy Mode should reduce complexity, not erase risk.
- Money, legal, client, public, medical, grant, and platform decisions still need approval.
- Every decision should create a log entry.
- If a decision creates work, ATLAS should create the related task or approval item.
