## Priority Scoring

The function returns a 0-100 score using these weights:

- `earlyProfit`: 25%
- `turnaroundSpeed`: 20%
- `longevity`: 20%
- `fundingPotential`: 15%
- `urgency`: 10%
- `personalImportance`: 10%

`buildDifficulty` subtracts a light penalty from the weighted score. A difficulty of 1 subtracts 0 points, and a difficulty of 10 subtracts 9 points.

## Priority Bands

- `Critical`: 80+
- `High`: 65-79.9
- `Medium`: 45-64.9
- `Low`: under 45

## Agent Router

`atlasAgentRouter` accepts a raw ATLAS command plus optional context flags and returns:

- `detectedDepartment`
- `assignedAgent`
- `confidence`
- `requiresColeApproval`
- `reason`
- `suggestedNextAction`
- `tags`

It routes grant and funding work to `ATLAS Grant Officer`, creator/client delivery work to `Creator Logistics Manager`, product/API/database work to `Product Manager`, outreach and pricing work to `Sales Assistant`, legal/finance work to `Legal/Finance Reviewer`, and personal planning work to `Chief of Staff`.

Cole approval is required for money movement, sending public or client messages, legal or financial decisions, external grant submissions, hiring, contracts, deleting data, and public posting.

## Long-Term Direction

ATLAS HQ should grow into the operating system for Avery Industries LLC: intake, routing, prioritization, task creation, drafting, technical dispatch, approval tracking, and daily or weekly planning. See `ATLAS_HQ_ARCHITECTURE.md` for the master direction and approval boundaries.
